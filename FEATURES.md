# ♟️ Blundermate 기능 및 구조 설명서 (AI -> User)

이 파일은 **AI(너)가 사용자(나)에게** 현재 프로젝트에 구현되어 있는 주요 기능과 파일 구조를 설명하고 문서화하는 곳입니다. 
새로운 기능이 추가되거나 구조가 변경될 때마다 AI가 이 파일을 업데이트하여 사용자에게 보고합니다.
---

본 애플리케이션은 크게 **홈 화면(초기 화면)**과 **분석 화면** 두 가지 뷰(View)로 나뉘어 동작합니다.

## 🏠 1. 홈 화면 (초기 화면)
사용자가 앱을 켰을 때 가장 먼저 만나는 화면으로, 분석할 게임을 입력하거나 저장된 기록을 열람하는 역할을 합니다.

- **게임 불러오기 (API 연동)**: Chess.com 사용자 이름을 입력하여 최근 10개의 게임 아카이브를 불러오고, 선택하여 바로 분석할 수 있습니다.
- **수동 PGN 입력**: 'Paste PGN manually' 버튼을 눌러 복사한 PGN 텍스트를 직접 붙여넣어 분석을 시작할 수 있습니다. 불완전한 PGN도 자체적으로 복구하여 파싱합니다.
- **체스판 직접 두기 (Play on board)**: 텍스트 기보가 없어도, 체스판 팝업 창 위에서 직접 기물을 움직여 게임을 만들고 곧바로 분석을 시작할 수 있습니다.
- **오답 노트 (My Vault)**: 분석 중 저장했던 실수(Blunder, Mistake 등) 목록을 카테고리/메모와 함께 열람하고, **연습 모드(Practice Mode)**에 진입하여 올바른 수를 직접 다시 찾아보는 복습이 가능합니다.
- **저장된 게임 (Saved Games)**: 분석이 완료되어 영구 저장된 전체 게임 목록을 확인하고, 클릭 한 번으로 분석 화면에 다시 불러올 수 있습니다.

## 🔍 2. 분석 화면 (Analysis View)
입력된 게임을 바탕으로 로컬 스톡피시 엔진이 가동되며, 실시간으로 기보를 분석하고 복기할 수 있는 핵심 화면입니다.

- **자동 엔진 분석 (Stockfish 18 Lite)**: 브라우저 Web Worker(`stockfish-18-lite-single.js`)를 통해 모든 수를 순차적으로 자동 분석하며, 승률(Win%) 변동과 Centipawn(CP) 손실을 계산하여 최선의 수부터 블런더(Blunder)까지 수를 등급별로 분류합니다. 호환성이 가장 뛰어난 Single-thread 버전을 사용하여 모바일에서도 안정적으로 구동됩니다.
- **인터랙티브 체스 보드 & 기보 테이블**: 상단에 체스 보드가 고정된 상태로 하단의 기보를 스크롤하며 클릭하거나, 버튼/단축키를 통해 수를 앞뒤로 탐색할 수 있습니다. 블런더나 실수 상황에서는 원래 둬야 했던 '최선의 수'를 보드 위에 시각적인 화살표로 그려줍니다.
- **자유 탐색 (Exploration) 모드**: 분석 중인 보드에서 기물을 다른 곳으로 직접 움직이면, 메인 분석이 일시 정지되고 해당 국면(가정의 수)에 대한 실시간 엔진 평가를 받아볼 수 있습니다.
- **엔진 추천 라인 시뮬레이션**: 'Top Engine Lines' 토글 영역에서 엔진이 추천하는 수(MultiPV) 중 하나를 클릭하면, 해당 라인이 체스판 위에서 자동으로 시뮬레이션 되어 이후의 변화를 눈으로 확인할 수 있습니다.
- **생성형 AI 해설 (Gemini)**: "✨ Gemini" 버튼을 누르면 창이 토글되며 해당 국면에 대한 해설을 제공할 수 있도록 준비되어 있습니다. (현재 UI 뼈대 구현 완료, API 연동 대기 중)
- **저장 기능 (Save)**: 현재 보고 있는 특정 수를 My Vault에 저장하거나, 분석된 전체 게임을 Saved Games에 영구 보관하는 모달 창 기능을 제공합니다.

## 🎨 3. 공통 UI/UX 및 최적화
- **100% 모바일 최적화**: 좁은 모바일 화면 및 하단 네비게이션 바(100dvh)에 완벽하게 대응하며, PC에서 접속해도 모바일 앱처럼 깔끔하게 중앙 정렬되어 표시되는 다크 모드 UI를 제공합니다.
- **반응형 보드 리사이징**: 기기 화면 높이가 부족할 경우 체스 보드 영역이 무너지지 않고 비례에 맞춰 부드럽게 조정됩니다.
- **퍼포먼스 렌더링 최적화**: 엔진의 잦은 콜백으로 인한 렌더링 병목 현상을 막기 위해 `requestAnimationFrame`과 스로틀링(Throttling)을 적용하여 부드러운 프레임레이트를 유지합니다.

## 📂 4. 프로젝트 파일 구조 (Project Structure)
코드의 가독성 및 유지보수성을 높이기 위해, 각 기능이 역할에 따라 철저히 모듈화(분리)되어 있습니다.
- **`index.html` & `styles.css`**: 앱의 전체 UI 뼈대 및 다크 모드 기반 반응형 모바일 뷰 스타일링
- **`main.js` (Controller)**: 전역 상태(State) 관리, 이벤트 리스너 연결, 화면 전환 등 애플리케이션의 전체 흐름 제어
- **`utils.js` (Logic)**: 엔진 평가값 파싱, 체스 기보 변환, 수학적 계산을 통한 국면 평가(Blunder 판별) 등 순수 연산 로직
- **`ui.js` (View)**: 복잡한 DOM 조작 및 화면 렌더링 전담 (기보 테이블, 오답노트 리스트, 평가 점수판 등)
- **`storage.js` (Model)**: 브라우저 `localStorage`를 활용한 데이터(My Vault, Saved Games)의 저장, 삭제, 불러오기 전담
- **`api.js`**: 외부 API (Chess.com 등)와의 네트워크 통신 및 데이터 가져오기 전담
- **`engine.js`**: 브라우저 백그라운드용 로컬 Stockfish Web Worker 구동 및 엔진 메시지 파싱(래퍼) 클래스
- **`engine/` 디렉토리**: Stockfish 엔진 코어 바이너리(`stockfish-18-lite-single.js`) 보관
